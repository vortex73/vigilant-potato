While venturing through the LLVM documentation, I encountered a reference made to SSA or Single Static Assignment forms. Many days,blogs and videos later, here's my understanding of SSA and how it changed the way I see code.

What is Static Single Assignment?

A program in which every variable is constrained to be assigned only once.

I would describe Static Single Assignment as a school of thought used to simplify representation of program constructs.

Understanding how the compilers understand programs

Resorting to a layman's definition of a compiler,

A compiler translates the high level source code into machine code.

Breaking down the compilation process there are mainly 4 processes that it carries out on the source code:

  • Lexing: Breaking down the source into "tokens".
  • Parsing: walking through the tokens and generating an Abstract Syntax Tree(AST).
  • Validation: checks if the AST is compliant with the language's rules
  • Translation: chews on the AST and spits out machine code.
